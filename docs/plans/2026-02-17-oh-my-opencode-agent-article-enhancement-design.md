## 三、设计方案

### 3.1 整体架构

采用**渐进式扩充**方案，在 1.1 节内部分三层：

```
### 1.1 Sisyphus: 智能项目经理

【第一层：基础介绍】(保留原有)
- 定位、核心能力
- 适用场景
- 实战案例
- 使用命令

---

【第二层：深度解析】(新增)
#### 🔧 Sisyphus 深度解析

- 工作流程：五阶段智能编排
- 推荐配置参数
- 并行执行优势：时间节省 33%
- 验证机制与核心原则

---

【第三层：UltraWork Mode】(新增)
#### ⚡ UltraWork Mode：一键全员激活

- 什么是 UltraWork Mode
- 使用方法
- 四大设计原则
- 适用场景对比
- ⚠️ 重要注意事项
- 验证清单
```

### 3.2 内容组织原则

**1. 层次递进**
- 第一层：快速上手，满足 80% 读者需求
- 第二层：深度理解，满足进阶用户
- 第三层：高级特性，满足专业用户

**2. 视觉区分**
- 用 emoji (🔧 ⚡) 标识新增部分
- 用 `---` 分隔符划分层次
- 用表格呈现复杂信息
- 用对比展示效率提升

**3. 风格一致**
- 保持原文的对比表格风格
- 保持实战案例说明
- 保持 emoji 点缀但不过度
- 保持简洁、引流型语言

### 3.3 关键设计决策

**决策 1：为何不拆分成独立章节？**
- ❌ 拆分会打破原有 6 个 Agent 并列结构
- ✅ Sisyphus 作为核心编排器，值得更多篇幅
- ✅ 内部分层既保证深度又不破坏结构

**决策 2：UltraWork Mode 为何放在 Sisyphus 内？**
- UltraWork Mode 是 Sisyphus 的高级特性，非独立 Agent
- 与 Sisyphus 的工作流程紧密相关
- 便于读者理解"何时用专业 Agent，何时用 UltraWork"

**决策 3：如何平衡深度与易读性？**
- 技术细节用表格压缩信息密度
- 抽象原理用实战案例解释
- 数据对比用具体数字（6分钟→4分钟）
- 避坑指南用 ❌ ✅ 清晰展示

## 四、内容详细设计

### 4.1 深度解析部分

**工作流程：五阶段智能编排**

采用表格形式，包含 4 列：
- 阶段编号（1️⃣ - 5️⃣）
- 名称（意图门槛、代码库评估等）

- 关键约束（加粗强调）

补充核心理念：职责分离是效率的关键

**推荐配置参数**

采用对比表格，包含 3 列：
- 配置项
- 推荐值
- 说明（重点用 🔥 标注）

重点强调：使用其他模型会导致体验显著下降

**并行执行优势**

采用数据对比表格：
- 串行执行：6 分钟（基准 100%）
- 并行执行：4 分钟（⚡ 节省 33% 时间）

补充工作原理（3 个要点）和实战案例

委托优先原则：
- 判断任务需要哪个专家
- 委托给最合适的 Agent
- 验证结果并整合

补充"何时自己动手"的边界条件

提供 2 种触发方式的代码示例
激活确认的 3 个标志

**四大设计原则**

采用表格形式，包含 3 列：
